# STM32 嵌入式硬件搭建保姆级教程

## 一、硬件清单（你需要购买/准备的东西）

### 核心板
| 序号 | 名称 | 型号 | 数量 | 参考价 | 备注 |
|------|------|------|------|--------|------|
| 1 | STM32 最小系统板 | STM32F103C8T6 | 1 | ~10元 | 俗称"蓝色药丸"(Blue Pill) |
| 2 | ST-Link V2 调试器 | ST-Link V2 | 1 | ~15元 | 烧录+调试用，USB 接电脑 |

### 传感器模块
| 序号 | 名称 | 型号 | 数量 | 参考价 | 备注 |
|------|------|------|------|--------|------|
| 3 | 超声波测距模块 | HC-SR04 | 1 | ~5元 | 4 针：VCC/Trig/Echo/GND |
| 4 | 红外避障模块 | 红外对管模块 | 2 | ~3元/个 | 左右各一个，3 针：VCC/OUT/GND |
| 5 | GPS 定位模块 | ATGM336H | 1 | ~20元 | 4 针：VCC/TX/RX/GND，自带天线 |
| 6 | GSM/GPRS 模块 | SIM900A | 1 | ~30元 | 短信+上网，需插 SIM 卡 |
| 7 | 摄像头模块 | OV7670（无FIFO版） | 1 | ~15元 | 18 针，接线最复杂 |
| 8 | 语音识别模块 | CI1303/LD3320 | 1 | ~25元 | 离线语音识别，4 针 UART 接口 |

### 执行器
| 序号 | 名称 | 型号 | 数量 | 参考价 | 备注 |
|------|------|------|------|--------|------|
| 9 | 震动马达 | 扁平振动马达 3V | 3 | ~2元/个 | 左/中/右三路，需 NPN 三极管驱动 |
| 10 | 有源蜂鸣器 | 3.3V 有源蜂鸣器 | 1 | ~2元 | 高电平响 |
| 11 | SOS 按键 | 轻触按键 | 1 | ~1元 | 普通按钮即可 |

### 辅材
| 序号 | 名称 | 数量 | 备注 |
|------|------|------|------|
| 12 | 面包板 | 1-2 块 | 830 孔大面包板推荐 |
| 13 | 杜邦线 | 若干 | 公对公、公对母、母对母各 20 根 |
| 14 | USB 数据线 | 1 根 | Micro-USB，给 STM32 板供电 |
| 15 | SIM 卡 | 1 张 | 普通大卡(SIM900A 用大卡)，需开通 GPRS |
| 16 | 锂电池 | 1 个 | 3.7V 锂电池（可选，也可 USB 供电调试） |
| 17 | NPN 三极管 | 3 个 | S8050 或 2N2222，驱动震动马达 |
| 18 | 电阻 | 若干 | 1KΩ（三极管基极）、10KΩ（按键上拉备用） |

**预计总成本：约 130-180 元**

## 二、接线图（最重要的部分）

### 总览：STM32F103C8T6 引脚分配

```
                    STM32F103C8T6 (Blue Pill)
                    ┌─────────────────────┐
            PA0  ───┤ 1              40 ├─── PB12  → 蜂鸣器
 超声波Trig PA1  ───┤ 2              39 ├─── PB13  → SOS 按键
 超声波Echo PA2  ───┤ 3              38 ├─── PB14  → OV7670 D5
    GPS TX  PA3  ───┤ 4              37 ├─── PB15  → OV7670 D6
    GPS RX  PA4  ───┤ 5              36 ├─── PA8   → OV7670 D0
  电池 ADC  PA5  ───┤ 6              35 ├─── PA9   → 语音模块 TX
OV7670 PCLK PA6  ───┤ 7              34 ├─── PA10  → 语音模块 RX
  马达左 PWM PA7  ───┤ 8              33 ├─── PA11  → OV7670 D3
 马达中 PWM PB0  ───┤ 9              32 ├─── PA12  → OV7670 D4
 马达右 PWM PB1  ───┤ 10             31 ├─── PA13  → SWDIO (调试)
  红外左    PB3  ───┤ 11             30 ├─── PA14  → SWCLK (调试)
  红外右    PB4  ───┤ 12             29 ├─── PA15  → OV7670 D7
 OV7670 D1  PB5  ───┤ 13             28 ├─── PB10  → SIM900A TX
 OV7670 D2  PB6  ───┤ 14             27 ├─── PB11  → SIM900A RX
OV7670 SCL  PB7  ───┤ 15             26 ├─── PB8   → OV7670 VSYNC
OV7670 SDA  PB9  ───┤ 16             25 ├─── 3.3V
OV7670 HREF        ├───── GND ─────────┤
                    └─────────────────────┘
```

### 第 1 步：连接 ST-Link 调试器（4 根线）

**这是第一步，连好后就能烧录程序。**

```
ST-Link V2          STM32 板
┌──────────┐        ┌──────────┐
│ SWDIO    ├────────┤ PA13     │ (丝印标 DIO 或 SWDIO)
│ SWCLK    ├────────┤ PA14     │ (丝印标 CLK 或 SWCLK)
│ GND      ├────────┤ GND      │
│ 3.3V     ├────────┤ 3.3V     │ (ST-Link 可以给板子供电)
└──────────┘        └──────────┘
```

> 操作：用 4 根母对母杜邦线连接。ST-Link 插电脑 USB 口。

### 第 2 步：连接超声波 HC-SR04（4 根线）

```
HC-SR04             STM32
┌──────────┐        ┌──────────┐
│ VCC      ├────────┤ 5V       │ (注意：HC-SR04 需要 5V!)
│ Trig     ├────────┤ PA0      │
│ Echo     ├────────┤ PA1      │
│ GND      ├────────┤ GND      │
└──────────┘        └──────────┘
```

> 注意：HC-SR04 需要 5V 供电，但 Echo 输出也是 5V。STM32 的 PA1 是 5V 容忍引脚，可以直接连。如果不放心，加一个 1KΩ + 2KΩ 分压电阻。

### 第 3 步：连接红外避障模块 x2（每个 3 根线）

```
左侧红外             STM32          右侧红外
┌──────────┐        ┌──────────┐  ┌──────────┐
│ VCC      ├────┬───┤ 3.3V     ├──┤ VCC      │
│ OUT      ├────│───┤ PB1      │  │ OUT      ├──→ PB3
│ GND      ├────┼───┤ GND      ├──┤ GND      │
└──────────┘    │   └──────────┘  └──────────┘
```

### 第 4 步：连接 GPS 模块 ATGM336H（4 根线）

```
ATGM336H            STM32
┌──────────┐        ┌──────────┐
│ VCC      ├────────┤ 3.3V     │
│ TX       ├────────┤ PA3 (RX) │  ← GPS 的 TX 接 STM32 的 RX!
│ RX       ├────────┤ PA2 (TX) │  ← GPS 的 RX 接 STM32 的 TX!
│ GND      ├────────┤ GND      │
└──────────┘        └──────────┘
```

> 关键：TX/RX 交叉连接！GPS 的 TX → STM32 的 RX (PA3)。

### 第 5 步：连接 SIM900A（4 根线 + SIM 卡）

```
SIM900A              STM32
┌──────────┐        ┌──────────┐
│ VCC(5V)  ├────────┤ 5V       │  ← 需要独立 5V 2A 电源!
│ TX       ├────────┤ PB11(RX) │  ← TX/RX 交叉!
│ RX       ├────────┤ PB10(TX) │
│ GND      ├────────┤ GND      │  ← 必须共地!
└──────────┘        └──────────┘
```

> 重要：SIM900A 峰值电流 2A，不能用 STM32 板上的 3.3V/5V 供电！需要独立电源（如 5V 2A 充电宝或电源适配器）。插入 SIM 卡后才能使用。

### 第 6 步：连接蜂鸣器（2 根线）

```
有源蜂鸣器            STM32
┌──────────┐        ┌──────────┐
│ + (正极)  ├────────┤ PB12     │
│ - (负极)  ├────────┤ GND      │
└──────────┘        └──────────┘
```

### 第 7 步：连接 SOS 按键（2 根线）

```
        STM32 PB13 ────┤按键├──── GND
```

> 代码里已配了内部上拉，按下按键接地触发中断。

### 第 8 步：连接震动马达 x3（需要三极管驱动）

每个马达的接线：
```
STM32 PWM引脚 ──[1KΩ]──→ 三极管(Base)
                         三极管(Emitter) → GND
                         三极管(Collector) → 马达(-)
                                           马达(+) → 3.3V
```

三路对应：
- 左马达：PA6 → 三极管 → 马达
- 中马达：PA7 → 三极管 → 马达
- 右马达：PB0 → 三极管 → 马达

### 第 9 步：连接语音识别模块 CI1303（4 根线）

```
CI1303               STM32
┌──────────┐        ┌──────────┐
│ VCC      ├────────┤ 3.3V     │
│ TX       ├────────┤ PA10(RX) │  ← TX/RX 交叉!
│ RX       ├────────┤ PA9 (TX) │
│ GND      ├────────┤ GND      │
└──────────┘        └──────────┘
```

### 第 10 步：连接 OV7670 摄像头（18 根线，最复杂）

```
OV7670               STM32
┌──────────┐        ┌──────────┐
│ 3V3      ├────────┤ 3.3V     │
│ GND      ├────────┤ GND      │
│ SCL(SIOC)├────────┤ PB6      │ I2C 时钟
│ SDA(SIOD)├────────┤ PB7      │ I2C 数据
│ VSYNC    ├────────┤ PB8      │
│ HREF     ├────────┤ PB9      │
│ PCLK     ├────────┤ PA5      │
│ D0       ├────────┤ PA8      │
│ D1       ├────────┤ PB4      │
│ D2       ├────────┤ PB5      │
│ D3       ├────────┤ PA11     │
│ D4       ├────────┤ PA12     │
│ D5       ├────────┤ PB14     │
│ D6       ├────────┤ PB15     │
│ D7       ├────────┤ PA15     │
│ XCLK     ├────────┤ (需外部晶振或MCO输出) │
│ PWDN     ├────────┤ GND (常低=工作)       │
│ RESET    ├────────┤ 3.3V (常高=不复位)    │
└──────────┘        └──────────┘
```

> 提示：OV7670 是最复杂的模块。**建议先不接摄像头**，先把其他模块调通。摄像头功能在毕设答辩时可以用 App 端的手机摄像头演示代替。

### 第 11 步：电池电压检测（可选）

```
锂电池 3.7V ──[10KΩ]──┬──[10KΩ]── GND
                       │
                       └──→ PA4 (ADC 采集分压后电压)
```

## 三、建议接线顺序（从简到繁）

**不要一次全接！按以下顺序逐步添加，每加一个模块就测试一次：**

1. **ST-Link + STM32 板** → 能烧录程序、LED 闪烁
2. **+ 蜂鸣器 + SOS 按键** → 按下按键蜂鸣器响
3. **+ 超声波** → 串口打印距离数据
4. **+ 红外避障 x2** → 手挡住时蜂鸣器/马达响
5. **+ 震动马达 x3** → 完整预警体验
6. **+ GPS** → 串口打印经纬度
7. **+ SIM900A** → 发短信、GPRS 连云端
8. **+ 语音模块** → 语音控制
9. **+ OV7670 摄像头** → 最后再搞，最复杂

## 四、软件环境配置

### 方案 A：Keil MDK（你已有）

1. 打开 Keil uVision5
2. 按 `embedded/MDK-ARM/README_KEIL_CONFIG.md` 创建工程
3. 编译 → 烧录 → 调试

### 方案 B：VS Code + PlatformIO（推荐，更现代）

你已有 VS Code，装个插件就能开发 STM32：

1. **安装 PlatformIO 插件**
   - VS Code 扩展商店搜索 `PlatformIO IDE`，安装
   - 重启 VS Code

2. **创建 PlatformIO 项目**
   - 点击 PlatformIO 图标 → New Project
   - Board: `genericSTM32F103C8`（BluePill）
   - Framework: `stm32cube`（HAL 库）

3. **复制代码**
   - 把 `embedded/Core/Src/*.c` 复制到 PlatformIO 的 `src/`
   - 把 `embedded/Core/Inc/*.h` 复制到 `include/`

4. **编译烧录**
   - 点击底栏的 ✓ 编译
   - 点击 → 烧录（需连接 ST-Link）

### 方案 C：STM32CubeMX + Keil（图形化配置）

1. 下载安装 [STM32CubeMX](https://www.st.com/en/development-tools/stm32cubemx.html)（免费）
2. 新建项目，选 STM32F103C8Tx
3. 按 `pin_config.h` 的引脚表在图形界面配置各外设
4. 生成 Keil 工程
5. 把本项目的 `.c/.h` 文件复制进去

## 五、烧录步骤

### 硬件连接确认
1. ST-Link 插电脑 USB
2. ST-Link 的 4 根线连到 STM32 板（SWDIO/SWCLK/3.3V/GND）
3. STM32 板上的电源 LED 亮起

### Keil 烧录
1. 编译通过（F7，0 Error）
2. `Options for Target` → `Debug` → 选 `ST-Link Debugger`
3. `Settings` → `Flash Download` → 勾选 `Reset and Run`
4. 点击 `Download`（F8）
5. 听到蜂鸣器响 2 声（启动提示）→ 再响 3 声（系统就绪）= 成功！

### VS Code + PlatformIO 烧录
1. 连好 ST-Link
2. 点击底栏 `→` (Upload) 按钮
3. 等待终端显示 `SUCCESS`

## 六、上电后的行为

烧录成功后，系统启动流程：

1. 蜂鸣器响 **2 声** → 表示开始初始化
2. 各模块依次初始化（超声波、GPS、红外、马达...）
3. SIM900A 初始化（等待约 3 秒）
4. 蜂鸣器响 **3 声** → 表示系统就绪

就绪后的自动行为：
- 每 200ms 测一次距离 → 障碍物 < 50cm 时蜂鸣器响 + 马达震动
- 每 1s 解析一次 GPS → 10s 上报一次位置到云端
- 按下 SOS 按键 → 发短信 + 蜂鸣器响 3 秒

## 七、常见问题排查

| 现象 | 可能原因 | 解决方法 |
|------|----------|----------|
| ST-Link 识别不到芯片 | 接线错误 | 检查 SWDIO/SWCLK 是否接反 |
| 烧录后无反应 | BOOT0 跳线帽位置不对 | BOOT0 接 GND（正常运行模式） |
| 蜂鸣器不响 | 正负极接反 | 换一下正负极 |
| 超声波数据异常 | Echo 没接对 | 确认 Echo → PA1 |
| GPS 无数据 | TX/RX 接反 | GPS TX → STM32 PA3(RX) |
| SIM900A 无响应 | 供电不足 | 换独立 5V 2A 电源 |
| SIM900A 报 SIM 卡错误 | SIM 卡没插好/没激活 | 重新插卡，确认卡有余额 |
| 马达不转 | 没接三极管 | 马达需要三极管驱动，不能直连 GPIO |

## 九、毕设答辩演示建议

如果时间紧张，**最小可演示方案**只需要：

1. **STM32 + ST-Link** → 基础运行
2. **超声波 + 蜂鸣器** → 演示避障预警（手放近蜂鸣器响）
3. **SOS 按键** → 演示紧急求助
4. **手机 App** → 演示地图、轨迹、视觉识别

这 4 样东西加起来不到 30 元，接线不超过 10 根，半小时搞定。
